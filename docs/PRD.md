# 產品需求文件（PRD）— 任務管理系統

## 1. 專案概述

### 1.1 背景與動機

在日常生活與工作學習中，每個人都需要管理各種待辦事項。然而，當任務數量增多時，容易遺忘或混淆優先順序，導致效率低下。本系統旨在提供一個簡單直覺的線上任務管理工具，幫助使用者有效追蹤與管理每日任務。

### 1.2 目標用戶

- **所有需要管理待辦事項的人**，包括學生、上班族、自由工作者
- 不限年齡與技術背景，介面設計力求簡單易用
- 適合個人使用的輕量級任務管理

### 1.3 核心價值主張

- **簡單直覺**：不需要學習成本，打開即可使用
- **一目了然**：清楚顯示任務狀態、優先程度與截止日期
- **高效管理**：快速新增、編輯、完成與刪除任務

## 2. 功能需求

### 2.1 主要功能清單

| 編號 | 功能名稱 | 說明 |
|------|----------|------|
| F01 | 任務列表 | 首頁顯示所有任務，可依狀態篩選 |
| F02 | 新增任務 | 填寫表單新增一筆任務 |
| F03 | 編輯任務 | 修改既有任務的內容 |
| F04 | 刪除任務 | 確認後移除一筆任務 |
| F05 | 標記完成 | 一鍵切換任務的完成/未完成狀態 |
| F06 | 任務詳情 | 查看單筆任務的完整資訊 |
| F07 | 狀態篩選 | 篩選顯示全部/未完成/已完成的任務 |
| F08 | 任務統計 | 顯示任務總數、已完成數、完成率 |

### 2.2 使用者故事

#### F01 — 任務列表
> 作為**一般使用者**，我希望**開啟網頁就能看到所有任務的列表**，以便**快速掌握目前有哪些待辦事項**。

#### F02 — 新增任務
> 作為**一般使用者**，我希望**透過表單輸入任務標題、描述、截止日期與優先等級**，以便**將新的待辦事項記錄到系統中**。

#### F03 — 編輯任務
> 作為**一般使用者**，我希望**點擊編輯按鈕後修改任務的標題、描述、截止日期或優先等級**，以便**更新任務內容反映最新需求**。

#### F04 — 刪除任務
> 作為**一般使用者**，我希望**點擊刪除按鈕後移除不再需要的任務**，以便**保持列表的整潔**。

#### F05 — 標記完成
> 作為**一般使用者**，我希望**一鍵將任務標記為已完成或恢復為未完成**，以便**快速更新任務進度**。

#### F06 — 任務詳情
> 作為**一般使用者**，我希望**點擊任務標題後查看完整的任務資訊**，以便**了解任務的所有細節**。

#### F07 — 狀態篩選
> 作為**一般使用者**，我希望**在列表頁透過按鈕篩選全部、未完成或已完成的任務**，以便**聚焦在特定狀態的任務上**。

#### F08 — 任務統計
> 作為**一般使用者**，我希望**在首頁看到任務總數、已完成數量與完成率**，以便**了解整體進度**。

## 3. 非功能需求

### 3.1 技術限制

| 項目 | 說明 |
|------|------|
| 後端框架 | Python + Flask |
| 模板引擎 | Jinja2（Flask 內建） |
| 資料庫 | SQLite（輕量級，無需額外安裝） |
| 前端樣式 | Bootstrap 5（CDN 引入） |
| 部署方式 | 本機開發環境（`flask run`） |

### 3.2 效能需求

- 頁面載入時間應在 **1 秒內**
- 資料庫操作（CRUD）應在 **500 毫秒內** 完成

### 3.3 安全考量

- 使用 Flask 內建的 CSRF 保護機制
- 所有使用者輸入需經過驗證，防止 SQL Injection
- 使用 Jinja2 的自動轉義功能防止 XSS 攻擊

### 3.4 可用性

- 介面設計簡潔直覺，初次使用者無需說明即可操作
- 支援桌面瀏覽器（Chrome、Firefox、Edge）
- 使用 Bootstrap 5 實現基本的響應式設計
